Директива rtSort

Данная директива предназначена для настройки сортировок списка. Директива принимает в качестве обязательного параметра идентификатор сортировки, который будет передаваться в запросе за данными.

Ниже расположен живой пример списка с сортировкам, а так же объявления контрактов SortParameter и AirportsListRequest. Моменты, на которые следует обратить внимание, следующие:

  • К элементу, помеченному данной директивой, добавляется класс rt-sortable. С помощью данного класса вы можете настроить стили, сигнализирующие, что элемент сортируемый. Например, в наших примерах это появление двунаправленной стрелки справа от элемента при наведении мыши.
  • При клике по элементу переданный параметр-идентификатор добавляется в коллекцию примененных сортировок с направлением Asc и тут же вызывается метод загрузки данных, в который передается массив sortings с указанной сортировкой в указанном направлении. К самому элементу добавляется класс rt-sort-asc, с помощью которого можно настилить внешний вид отсортированного столбца.
  • При повторном клике направление сортировки меняется на противоположное и выполняется запрос за данными. Класс rt-sort-asc элемента изменяется на rt-sort-desc.
  • С помощью клика с зажатой клавишей Ctrl можно настроить сортировку по нескольким полям. В таком случае, в массив sortings будет передаваться несколько значений.

Компонент rt-row-number

Данный компонент может быть использован в шаблонах элемента списка для автоматической нумерации строк. Компонент автоматически распознает с каким пейджером он работает и проставляет соответствующим образом номера, например, начиная нумерацию не с 1, а с 21 для второй страницы постраничного списка.

Данная директива принимает обязательный параметр index, на основе которого и будет считаться номер строки.

Директивы rtLoadData, rtReloadData, rtCancelLoad, rtResetSettings

Данные директивы позволяют добавить к HTML-элементам следующий функционал:

  • rtLoadData - при клике на элемент, помеченный данной директивой, будет выполняться загрузка данных в список.
  • rtReloadData - при клике на элемент, помеченный данной директивой, будет выполняться перезагрузка (загрузка со сбросом параметров пейджинга) данных в список.
  • rtCancelLoad - при клике на элемент, помеченный данной директивой, будет происходить отмена выполняющегося в данный момент времени запроса данных для списка.
  • rtResetSettings - при клике на элемент, помеченный данной директивой, будет выполняться сброс параметров списка (сортировки, пейджинг, фильтры).

Так же данные директивы выставляют значения для свойства disabled в зависимости от текущего состояния списка. Например, блокируя кнопку загрузки, если запрос за данными уже выполняется.
